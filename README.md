# UI Review and Responsive Web Design

# About the Project
This project was created to understand how real-world frontend layouts work and how to fix common UI issues. Instead of just building a webpage, the focus was on improving layout structure, fixing alignment problems, and making the design responsive across different screen sizes.
The goal was to practice writing clean HTML and CSS and to understand how small changes in layout and styling can improve overall user experience.

# What I Built
- A simple landing page with header, hero section, features section, and footer  
- Structured the page using semantic HTML for better readability  
- Designed layouts using Flexbox  
- Made the page responsive for mobile and desktop using media queries  

## Key Improvements Focused
- Fixed alignment and spacing issues  
- Ensured consistent layout across sections  
- Improved responsiveness for smaller screens  
- Made the UI clean and easy to read  

## Technologies Used
- HTML5  
- CSS3 (Flexbox, Media Queries)

## What I Learned
- How to structure HTML properly using semantic elements  
- How Flexbox helps in building flexible layouts  
- How to make a website responsive using media queries  
- How to identify and fix UI issues like broken layouts and poor spacing  

## Final Thoughts
This project helped me understand that frontend development is not just about building UI, but also about improving and refining it. Even simple layouts can be optimized for better usability and responsiveness.
